## Project Overview

Engineering Department Daily Update Dashboard вЂ” a React dashboard for tracking team work hours, analyzing consistency, and surfacing risk warnings. UI is in Traditional Chinese.

## Architecture

- **`index.html`** (~1200 lines): React 18 + Recharts loaded via unpkg CDN with Babel Standalone for in-browser JSX transpilation. `Dashboard` component fetches `raw_data.json` and `gitlab-commits.json` at runtime with loading/error states.
- **`raw_data.json`**: Dashboard data вЂ” `rawData` (work hours by date/member) and `issues` (risk warnings). Members list and colors are auto-computed from rawData keys.
- **`gitlab-commits.json`**: GitLab commit data вЂ” `commits` (by date/member with URLs), `analysis` (consistency status), `projectRisks` (single-contributor warnings). Generated by `scripts/fetch-gitlab-commits.js`. Dashboard gracefully hides commit features if this file is missing.
- **`chat-config.json`** (gitignored): Google Chat space config with `spaceId` and `memberMap` for user ID to name mapping.
- **`gitlab-config.json`** (gitignored): GitLab API config with `baseUrl`, `token` (PAT with `read_api` scope), `memberMap` (GitLab author в†’ daily update member name), and `excludeAuthors`.
- **`scripts/parse-daily-updates.js`**: Shared parsing logic for daily update data вЂ” hour extraction, leave detection, issue generation. Used by both `/fetch-daily-updates` and `/backfill-daily-updates` skills. Run via `node scripts/parse-daily-updates.js <messages-file> [--leave "Name:M/D-M/D"]`.
- **`scripts/merge-daily-data.js`**: Merges parsed daily update output with existing raw_data.json data. Used by `/sync-daily-updates` skill.
- **`scripts/fetch-gitlab-commits.js`**: Fetches commits from self-hosted GitLab (`biglab.buygta.today`), maps authors to members, runs consistency analysis against `raw_data.json`, writes `gitlab-commits.json` and outputs POST payload to stdout. Run via `node scripts/fetch-gitlab-commits.js --date <M/D>`.
- **`appscript/Code.gs`** + **`appscript/index.html`**: Google Apps Script web app serving Sheets-based dashboard. `index.html` must stay in sync with the main `index.html` (data loading uses `google.script.run` instead of `fetch`). Always update both when modifying the dashboard.
- Four dashboard views: Daily, Trend, Weekly, Commits
- Dark theme with inline styles; no external CSS; teal (`#06b6d4`) accent for commit-related UI

## Development

**To run locally**: Serve with `python3 -m http.server` (fetch requires HTTP, not file://). Open `http://localhost:8000`.

**To run tests**: `npm test` (Vitest, 48 tests). Tests validate `raw_data.json` schema, `SEVERITY_COLORS` mapping, and `fetch-gitlab-commits.js` utilities.

**To update data manually**: Edit `raw_data.json` directly. Charts and tables render reactively from this data.

**To update data from Google Chat**: Run `/sync-daily-updates` skill вЂ” fully automated: fetches Chat messages, parses hours, merges data, commits, pushes, and optionally updates Google Sheets.

**To update GitLab commits**: Run `/sync-gitlab-commits` skill вЂ” fetches commits from GitLab, analyzes consistency with daily updates, writes to Spreadsheet and local JSON.

**To sync everything**: Run `/sync` skill вЂ” runs both `/sync-daily-updates` and `/sync-gitlab-commits` in parallel.

## Google Chat Integration

The team posts daily work hour reports in Google Chat space `spaces/AAQAQhmoRAk`. The `/sync-daily-updates` skill automates data extraction:

1. Reads `chat-config.json` for space ID and member mapping
2. Fetches messages via `mcp__gws__chat_spaces_messages_list`
3. Saves output to file, runs `node scripts/parse-daily-updates.js` to parse
4. Script extracts hours, detects leave, generates issues
5. Thread date в‰  content date ("3/6 Daily Update" contains 3/5 progress)
6. Reviews output, merges into `raw_data.json`, and runs tests

The older `/fetch-daily-updates` and `/backfill-daily-updates` skills are superseded by `/sync-daily-updates`.

## GitLab Integration

Self-hosted GitLab at `https://biglab.buygta.today`. The `/sync-gitlab-commits` skill automates commit data extraction:

1. Reads `gitlab-config.json` for API token and member mapping (14 members, 5 excluded bots/non-eng)
2. Fetches commits via GitLab API (`/api/v4/projects/:id/repository/commits?all=true`)
3. Maps `author_name` to daily update member names; unmapped authors logged as warnings
4. Cross-references with `raw_data.json` for consistency analysis (вњ… both present, вљ пёЏ hours but no commits, рџ”ґ commits but no report)
5. Identifies single-contributor project risks
6. Writes `gitlab-commits.json` (dashboard) + POSTs to Apps Script (Spreadsheet)
7. Each commit includes a clickable `url` linking to the GitLab commit page
