# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Engineering Department Daily Update Dashboard — a React dashboard for tracking team work hours, analyzing consistency, and surfacing risk warnings. UI is in Traditional Chinese.

## Architecture

- **`index.html`** (~15 lines): Vite entry point. Loads Google Fonts and mounts `<div id="root">` with `<script type="module" src="/src/main.tsx">`.
- **`src/`**: TypeScript React source (Bun + Vite build):
  - `main.tsx` — GitHub Pages entry (fetch-based data loading)
  - `main.appscript.tsx` — Apps Script entry (google.script.run loading)
  - `App.tsx` — Main Dashboard component (~170 lines), accepts `loadData` prop, state + routing
  - `views/` — View components: `DailyView.tsx`, `TrendView.tsx`, `WeeklyView.tsx`, `StatusOverview.tsx`
  - `hooks/` — Custom hooks: `useCurrentWeek`, `useDailyBarData`, `useTrendData`, `useWeeklySummary`, `useAllIssues`
  - `CommitsView.tsx` — Commits tab with scatter chart, consistency grid, task warnings
  - `components.tsx` — Shared UI (CustomTooltip, CardPanel, ColorDot, StatusBadge)
  - `constants.ts` — COLORS, SEVERITY_COLORS, THRESHOLDS, palettes
  - `utils.ts` — Utility functions (date math, status calculation)
  - `types.ts` — TypeScript type definitions
  - `styles.css` — Global CSS (animations, responsive breakpoints)
  - `google.script.d.ts` — Type declarations for Apps Script
- **`public/raw_data.json`**: Dashboard data — `rawData` (work hours by date/member) and `issues` (risk warnings). Members list and colors are auto-computed from rawData keys.
- **`public/gitlab-commits.json`**: Commit data from both GitLab and GitHub — `commits` (by date/member with URLs), `analysis` (consistency status), `projectRisks` (single-contributor warnings, data generated but not displayed in dashboard). Each commit item includes a `source` field (`"gitlab"` or `"github"`); dashboard gracefully handles missing `source` (defaults to `"gitlab"`). Generated by `scripts/fetch-gitlab-commits.js` and/or `scripts/fetch-github-commits.js`. Dashboard gracefully hides commit features if this file is missing.
- **`public/task-analysis.json`**: Task reasonableness analysis — AI-generated warnings comparing daily update tasks against GitLab commits. Loaded by CommitsView to display task warnings. Generated automatically by `/sync` Stage 3 (`prepare-task-analysis.js` piped to `claude --print`), or manually. Dashboard gracefully hides task warnings if this file is missing.
- **`public/plan-analysis.json`**: Plan/spec file tracking analysis — AI-generated correlations between spec commits and daily updates. Loaded by PlanSpecView to display spec tracking. Generated by `/sync` Stage 4 (`detect-plan-specs.js` + `prepare-plan-analysis.js` piped to `claude --print`). Dashboard gracefully hides plan tracking features if this file is missing.
- **`chat-config.json`** (gitignored): Google Chat space config with `spaceId` and `memberMap` for user ID to name mapping.
- **`gitlab-config.json`** (gitignored): GitLab API config with `baseUrl`, `token` (PAT with `read_api` scope), `memberMap` (GitLab author → daily update member name), and `excludeAuthors`.
- **`github-config.json`** (gitignored): GitHub API config with `baseUrl`, `org`, `token` (PAT with `repo` scope), `memberMap` (GitHub username → daily update member name), and `excludeAuthors`.
- **`scripts/merge-daily-data.js`**: Merges parsed daily update output with existing `public/raw_data.json` data. Used by `/sync-daily-updates` skill.
- **`scripts/fetch-gitlab-commits.js`**: Fetches commits from self-hosted GitLab (`biglab.buygta.today`), maps authors to members, runs consistency analysis against `public/raw_data.json`, writes `public/gitlab-commits.json` and outputs POST payload to stdout. Run via `node scripts/fetch-gitlab-commits.js --date <M/D>`. Also exports shared functions (`collectCommits`, `buildAnalysis`, `buildDashboardJSON`, `buildPostPayload`) for use by other scripts.
- **`scripts/collect-gitlab-commits.js`**: Collection-only script — fetches commits and maps authors without running analysis. Outputs raw commits JSON to stdout. Used by `/sync` DAG Stage 1 for parallel collection. Run via `node scripts/collect-gitlab-commits.js --date <M/D>`.
- **`scripts/fetch-github-commits.js`**: Full GitHub pipeline — fetches commits from github.com org, deduplicates against existing GitLab data by SHA, runs consistency analysis, writes merged `public/gitlab-commits.json` and outputs POST payload. Run via `node scripts/fetch-github-commits.js --date <M/D>`. Exports `collectGitHubCommits` for use by collect script.
- **`scripts/collect-github-commits.js`**: Collection-only script for GitHub — fetches commits from github.com org. Outputs raw commits JSON to stdout. Used by `/sync` DAG Stage 1 for parallel collection. Run via `node scripts/collect-github-commits.js --date <M/D>`.
- **`scripts/analyze-consistency.js`**: Analysis-only script — reads commits JSON + `public/raw_data.json`, produces consistency analysis, merges into `public/gitlab-commits.json`. Used by `/sync` DAG Stage 2. Run via `node scripts/analyze-consistency.js --commits <path>`.
- **`scripts/prepare-task-analysis.js`**: Extracts task descriptions from daily update messages and cross-references with GitLab commit data. Generates a structured prompt for Claude to evaluate task reasonableness (hours vs output). Run via `node scripts/prepare-task-analysis.js --date <M/D>`. Output goes to stdout as a Claude prompt. Used by `/sync` DAG Stage 3 piped to `claude --print`.
- **`scripts/detect-plan-specs.js`**: Detects plan/spec documentation files in commits. Filters commit titles by keywords (plan, spec, design, docs, RFC, architecture), fetches file changes from GitLab/GitHub API for candidates, filters to doc paths (docs/, specs/, plans/, design/). Run via `node scripts/detect-plan-specs.js --date <M/D>`. Outputs JSON to stdout. Exports `matchesSpecKeyword`, `isDocFile`, `filterSpecCommits` for testing.
- **`scripts/prepare-plan-analysis.js`**: Builds Claude CLI prompt from detected spec commits + daily update text. Cross-references spec file commits with daily update work items. Run via `node scripts/prepare-plan-analysis.js --date <M/D> --specs <path>`. Output piped to `claude --print`. Exports `buildPrompt` for testing.
- **`appscript/index.html`**: Auto-generated by `bun run build:appscript` (gitignored). Do NOT edit manually.
- **`appscript/Code.gs`**: Google Apps Script server-side code. Deploy via `bun run deploy:appscript`.
- **`appscript/.clasp.json`** (gitignored): clasp config pointing to Apps Script project ID.
- **`appscript/appsscript.json`** (gitignored): Apps Script manifest (timezone, runtime, webapp settings).
- Five dashboard views: Daily, Trend, Weekly, Commits, Plan Tracking
- Dark theme with inline styles + `src/styles.css`; teal (`#06b6d4`) accent for commit-related UI

## Development

**To run locally**: `bun run dev` (Vite dev server with HMR). Open `http://localhost:5173`.

**To build**: `bun run build` (outputs to `dist/`).

**To build Apps Script version**: `bun run build:appscript` (generates `appscript/index.html`).

**To deploy Apps Script**: `bun run deploy:appscript` (builds + clasp push).

**To run tests**: `bun run test` (Vitest, 139 tests). Tests cover: data schema validation, script utilities, React hooks (unit), view components (rendering), consistency analysis logic, and deduplication. Note: use `bun run test` not `bun test` — the latter uses Bun's built-in runner which doesn't support jsdom.

**To update data manually**: Edit `public/raw_data.json` directly. Charts and tables render reactively from this data.

**To update data from Google Chat**: Run `/sync-daily-updates` — fully automated: fetches Chat messages, parses hours, merges data, commits, pushes, and updates Google Sheets.

**To update GitLab commits**: Run `/sync-gitlab-commits` — fetches commits from GitLab, analyzes consistency with daily updates, writes to Spreadsheet and local JSON.

**To update GitHub commits**: Run `/sync-github-commits` — fetches commits from GitHub, deduplicates against GitLab data, analyzes consistency, writes to Spreadsheet and local JSON.

**To sync everything**: Run `/sync` — 4-stage DAG pipeline: Stage 1 collects daily updates + GitLab/GitHub commits in parallel, Stage 2 runs consistency analysis, Stage 3 runs automated task analysis via `claude --print`, Stage 4 detects plan/spec files and runs plan tracking analysis via `claude --print`.

## Slash Commands

| Command | Description |
|---------|-------------|
| `/sync` | 四階段 DAG pipeline: 平行收集 → 一致性分析 → 任務合理性 → 規劃文件追蹤（主要使用） |
| `/sync-daily-updates` | Google Chat daily updates: 抓取 → 解析 → 合併 → commit → push → Sheets |
| `/sync-gitlab-commits` | GitLab commits: 抓取 → 分析一致性 → 寫入 JSON + Sheets。支援 backfill: `/sync-gitlab-commits 3/9-3/12` |

Skills 定義在 `.claude/skills/*.md`，需要 `user_invocable: true` frontmatter。

## Google Chat Integration

The team posts daily work hour reports in Google Chat space `spaces/AAQAQhmoRAk`. The `/sync-daily-updates` skill automates data extraction:

1. Reads `chat-config.json` for space ID and member mapping
2. Fetches messages via `mcp__gws__chat_spaces_messages_list`
3. Saves output to file, runs `node scripts/parse-daily-updates.js` to parse
4. Script extracts hours, detects leave, generates issues
5. Thread date ≠ content date ("3/6 Daily Update" contains 3/5 progress)
6. Reviews output, merges into `public/raw_data.json`, and runs tests

## GitLab Integration

Self-hosted GitLab at `https://biglab.buygta.today`. The `/sync-gitlab-commits` skill automates commit data extraction:

1. Reads `gitlab-config.json` for API token and member mapping (22 author variants → 14 members, 6 excluded bots/non-eng)
2. Fetches all visible projects with recent activity via `last_activity_after` filter (not just membership — covers `dailyview/*`, `bigdata/*`, `CrawlersV2/*`, `sinyi/*`, personal repos, etc.)
3. For each project, fetches commits via GitLab API (`/api/v4/projects/:id/repository/commits?all=true`)
4. Maps `author_name` to daily update member names; unmapped authors logged as warnings — some members use multiple git names (e.g., `byron.you`, `Byron.you`, `游日銜` all map to 日銜)
5. Cross-references with `public/raw_data.json` for consistency analysis (✅ both present, ⚠️ hours but no commits, 🔴 commits but no report)
6. Identifies single-contributor project risks
7. Writes `public/gitlab-commits.json` (dashboard) + POSTs to Apps Script (Spreadsheet)
8. Each commit includes a clickable `url` linking to the GitLab commit page and `source: "gitlab"`
9. Commits are deduplicated by `sha|project` across platforms — same commit appearing on both GitLab and GitHub is counted once

## GitHub Integration

GitHub.com org `bigdata-54837596`. The `/sync-github-commits` skill automates commit data extraction:

1. Reads `github-config.json` for API token, org, and member mapping
2. Fetches all repos in the org via GitHub REST API (`/orgs/{org}/repos`)
3. For each repo, fetches commits with date range filtering
4. Maps `author.login` to daily update member names; unmapped authors logged as warnings
5. SHA truncated to 8 characters to match GitLab's `short_id` format
6. Cross-references with existing GitLab data for SHA dedup (`sha|project` key)
7. Writes merged `public/gitlab-commits.json` (dashboard) + POSTs to Apps Script (Spreadsheet)
8. Each commit includes `source: "github"` and clickable `url` linking to the GitHub commit page

## Key Conventions

- Dependencies managed via Bun (React 18.2.0, Recharts 2.12.7, TypeScript 5)
- Color palette: background `#0f172a`, cards `#1e293b`, accents blue/green/yellow/red/orange, teal `#06b6d4` for commits
- `appscript/index.html` is auto-generated — never edit manually. Run `bun run build:appscript` to regenerate.
- Font stack: JetBrains Mono, SF Mono, Noto Sans TC
- Issue severity: 🔴 critical, 🟡 warning, 🟠 caution, 🟢 improvement
- `SEVERITY_COLORS` object maps severity emoji to `{ sc, bg }` color pair
